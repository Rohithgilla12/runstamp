# Impeccable — Design Context

> The single source of truth for visual + interaction design on Runstamp.
> Consulted on every UI change. Read this before adding a new screen,
> changing a color, picking a font weight, or writing copy that ships.

## Design Context

### Users

Vegetarian / serious runners who log 60–80 km/week across 4–5 sessions, often
travel and run in new cities, post selectively to Instagram Stories after long
runs, and have opinions about MAF, shoe rotations, and pace discipline. They
already use Strava or Apple Watch as their *tracker* — Runstamp is the
*post-run* tool they reach for to (a) compose a beautiful share image, (b)
see "the bigger picture" of where they've run and what they've earned, AND
(c) **read their training honestly** — VO₂ max trend, cadence, MAF aerobic
pace, decoupling, GAP, form / fitness / fatigue / TSB.

That third bucket is the one that's easy to under-design. These are runners
with opinions about training; they want the analytical surfaces to help them
*make decisions*, not just look pretty. "Quiet" is not the same as "thin on
data." A polished page that hides the number they came for is a failed page.

When they open Runstamp the run is already done. They want closure on share
surfaces, and **clarity** on analytical surfaces.

### Brand Personality

**Quiet · earned · collectible.**

The PRD calls this "quiet competence" — a well-designed tool that respects the
user. Voice is plain, runner-coded, slightly dry.

- **Good** — *"Sub-3:45 marathon. Stamped." / "47 cities. 6 countries."*
- **Bad** — *"AMAZING WORK!! 🔥🔥 You CRUSHED IT!!" / "Running rockstar 🚀"*

The app whispers. The run did the work. The stamp is the keepsake.

### Aesthetic Direction

**Postage / passport / postmark world.** No other running app occupies this
lane and we should keep it.

Reference triangle the project draws from:

| Source | What we borrow |
| --- | --- |
| **Things 3 / Bear / Linear** | Indie-grade craft. Restrained type doing the heavy lifting. No marketing-speak in UI. Snap-to-grid spacing. |
| **Apple Wallet / Mail.app passes** | Layered card surfaces. Real-world metaphor (boarding pass, ticket, passport entry). Stack of designed artifacts the user can flip through. |
| **Polaroid / Field Notes / Risograph print shops** | Tangible, slightly imperfect, smudged ink character. Halftone dots, perforated edges, postmark circles, air-mail stripes. |

Anti-references (do **not** look like these):

- **Duolingo** — gamified, over-emoji'd, cute mascot energy.
- **Nike Run Club** — aspirational sloganeering, cinematic black backgrounds with massive type.
- **Strava Premium** — analytics behind a paywall, generic share cards.

| Token | Light | Dark | Use |
| --- | --- | --- | --- |
| `paper` | `#f3ede2` | `#0e0d0b` | Canvas |
| `paper2/3` | `#ebe3d3 / #e2d8c4` | `#1a1714 / #22201c` | Card surfaces, hover/depth |
| `ink` | `#14110d` | `#f3ede2` | Primary type |
| `ink2/3` | `#3a342b / #75695a` | `#c9c0b1 / #8a8170` | Secondary type, hints |
| `accent` (solar) | `#e85d2f` | `#e85d2f` | One warm pop per surface; PRs, earned-stamp marks, share CTAs |
| `moss` | `#4a6b3a` | `#7a9a6a` | Positive deltas, "synced" dots |
| `line` / `line2` | rgba ink at 10/6% | rgba paper at 12/6% | Hairline dividers; never solid black |

Accent is user-selectable (`solar | electric | moss | ink`) but **solar is the
default and the brand color**. Other accents exist for runners who prefer a
calmer surface.

#### Type (already wired in `design/typography.tsx`)

- **Instrument Serif** *italic* — display moments. Section heroes ("The *bigger* picture."), card titles, stamp names.
- **Geist** — UI body, labels, secondary text. Most weight goes here.
- **JetBrains Mono** — numerals, eyebrows (`STAMP`, `LIFETIME`, etc.), monospaced metric blocks.

Numbers are sacred — never set a metric in a sans-serif. A distance, a pace,
a stamp count is always JetBrains Mono with negative letter-spacing.

#### Motifs (the philately vocabulary)

- Perforated edges on stamp-style cards
- Postmark circles (date / city around a rim)
- Halftone dot patterns over photos
- Air-mail diagonal stripes for accent borders on travel-themed templates
- Slight ink-bleed character (StampBadge, postmark rims) — never crisp vector

Every share-card template (Postage, Postmark, Boarding Pass, Passport,
Customs, Engraved, Wax Seal, plus future Halftone/Riso/Cyanotype/Date Stamp/
Minimal) lives inside this vocabulary.

### Surface taxonomy

Two kinds of surface, and they have different rules:

- **Artifact surfaces** — Home post-run card, Activity hero, Editor, all 12
  share-card templates, the Year-in-Stamps recap. These are *keepsakes*.
  Tangible, paper-grade, slow-read. The run is closed. Quiet rules.

- **Analytical surfaces** — Stats tab, Activity charts (HR / Pace / Splits),
  Best Efforts, Form / VO₂ / MAF cards, Places list, Stamps catalogue. These
  are *decision-support*. A runner is asking "am I drifting?", "did I build
  this month?", "did I dose effort right?". The chart, the number, the
  comparison is the whole point.

Rules 1–5 below apply to both. Rule 6 specifically governs analytical
surfaces and supersedes the hero-metric template the artifact family uses.

### Design Principles

These are the six rules to consult any time a design decision is in question.
In order of precedence — earlier rules win when they conflict.

1. **Quiet over loud.** The run did the work. The UI doesn't celebrate, it
   stamps. No exclamation marks, no emoji in product copy, no "AMAZING"
   adjectives. Numbers and serifs do the talking.

2. **Real-world materials.** Every surface borrows from print culture —
   perforations, postmarks, halftone dots, ink bleed, air-mail stripes. If a
   new component can't trace its visual language back to something tangible,
   it doesn't belong. (Applies less strictly to analytical surfaces, which
   are scientific-instrument-coded — clean ledger lines, not postage.)

3. **Type carries the load.** Instrument Serif italic for moments,
   JetBrains Mono for numerals, Geist for everything else. Never reach for
   illustrative chrome where a well-set number will do. No decorative
   borders, no gradients-for-the-sake-of-it, no drop shadows on text.

4. **One warm pop per surface.** Solar (`#e85d2f`) is the only saturated thing
   on the screen at any given moment. Use it for PRs, earned stamps, primary
   share CTAs — never for chrome, navigation states, or filler. Everything
   else is paper, ink, moss.

5. **Empty states look like the rest of the app.** No placeholder kitsch, no
   stock illustrations of generic runners, no "0 runs · let's go!" cheer.
   Honest copy: *"No runs yet — connect a source to stamp your first run."*

6. **Information answers a question; decoration doesn't.** Add chart chrome
   only when it answers a question a runner actually has. A scrub tooltip
   answers "what was the value at minute 42?" A compare legend answers
   "which bar is which when both periods are on screen?" A hero metric
   repeated under the chart answers nothing — the chart already shows it.

   On **analytical surfaces**, the chart is the hero, not a number above it.
   Y-axis tick labels and reference lines do the labeling that the artifact-
   family hero-metric template would otherwise duplicate. The runner came
   for the *shape*; the numbers are the shape's labels. If you find yourself
   showing the same value twice (a big AVG above the chart AND an AVG tile
   below it), delete one — usually the big one.

   On **artifact surfaces** (share cards, the Editor canvas, Home post-run),
   the hero-metric template is fine. The number is the brag; the chart is
   garnish. Different surface, different rule.

### Accessibility bar

WCAG AA, no special accommodations.

- AA contrast on all text (the ink/paper palette already clears this with room).
- 44pt minimum tap targets on actionable controls.
- Screen-reader labels on icon-only buttons (`accessibilityLabel`).
- Don't rely on color alone for state — pair the solar accent with text or
  iconography (already the pattern for PRs, earned stamps, etc.).
